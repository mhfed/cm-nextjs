```markdown
# Git Workflow

## Branch Naming

- feature/ (Feature branch / Nhánh tính năng)
- bugfix/ (Bug fix branch / Nhánh sửa lỗi)
- hotfix/ (Hotfix branch / Nhánh sửa lỗi khẩn cấp)
- release/ (Release branch / Nhánh phát hành)

## Commit Messages / Quy ước Commit

Follow Conventional Commits / Tuân thủ Conventional Commits:

- feat: New feature / Tính năng mới
- fix: Bug fix / Sửa lỗi
- docs: Documentation / Tài liệu
- style: Formatting / Định dạng
- refactor: Code restructuring / Tái cấu trúc code
- test: Testing / Kiểm thử
- chore: Maintenance / Bảo trì

## Pull Requests / Pull Requests

1. Create feature branch / Tạo nhánh tính năng
2. Implement changes / Thực hiện thay đổi
3. Write tests / Viết test
4. Create PR / Tạo PR
5. Code review / Review code
6. Merge / Merge

## Code Review Process / Quy trình Review Code

- Review checklist / Danh sách kiểm tra
- Performance considerations / Cân nhắc về hiệu năng
- Testing requirements / Yêu cầu về testing
```
