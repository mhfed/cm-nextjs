```markdown
# Code Style Guide / Hướng dẫn Code Style

## TypeScript

- Use strict mode / Sử dụng strict mode
- Define types for all props / Định nghĩa type cho tất cả props
- Avoid any / Tránh sử dụng any

## Components / Components

- Use functional components / Sử dụng functional components
- Props interface for each component / Interface props cho mỗi component
- Extract complex logic to hooks / Tách logic phức tạp vào hooks

## Naming Conventions / Quy ước đặt tên

- PascalCase for components / PascalCase cho components
- camelCase for functions / camelCase cho functions
- kebab-case for files / kebab-case cho files

## File Organization / Tổ chức file

- One component per file / Một component một file
- Group related components / Nhóm các component liên quan
- Shared utilities in lib/ / Các tiện ích dùng chung trong lib/
```
